# Documentation

This folder contains all project documentation in markdown format.

## Documentation Structure

- **Technical Specifications** - Detailed technical requirements and architecture
- **API Documentation** - Code API references and usage examples  
- **Game Design Documents** - Game mechanics, features, and design decisions
- **Development Guides** - Setup, building, and contribution guidelines
- **Architecture Decisions** - Record of important technical decisions
- **User Guides** - End-user instructions and gameplay help

## Getting Started

For developers new to this project:
1. Read the main [project README](../README.md)
2. Review the [project plan](../plan/plan.md)
3. Check the [development setup guide](setup.md) (when available)

## Contributing to Documentation

- Use clear, concise markdown formatting
- Include code examples where relevant
- Keep documentation up-to-date with code changes
- Follow the established folder structure
- Reference Phaser.io documentation for game engine specifics
